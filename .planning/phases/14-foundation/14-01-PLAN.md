**1. Create `src/hooks/useBreakpoint.ts`:**

Export `DeviceTier` type as `'mobile' | 'tablet' | 'desktop'`.

Export `getTier()` function that reads `window.matchMedia` synchronously:
- If `(min-width: 1024px)` matches → `'desktop'`
- Else if `(min-width: 768px)` matches → `'tablet'`
- Else → `'mobile'`

Export `useBreakpoint()` hook:
- Initialize `useState<DeviceTier>(getTier)` (function form — reads matchMedia synchronously, no flash)
- In `useEffect`, create two `MediaQueryList` objects: `window.matchMedia('(min-width: 768px)')` and `window.matchMedia('(min-width: 1024px)')`
- Attach `addEventListener('change', handleChange)` on both where `handleChange = () => setTier(getTier())`
- Return cleanup removing both listeners (use the same `const` references captured in effect body — do NOT create new matchMedia objects in cleanup)
- Return the tier string

Use `addEventListener('change', ...)` form, NOT the deprecated `addListener()`.

No debouncing needed — matchMedia fires only at defined breakpoints.

**3. Add matchMedia mock to `src/test/setup.ts`:**

Append after the existing `import '@testing-library/jest-dom';` line:

```typescript
import { vi } from 'vitest';

Object.defineProperty(window, 'matchMedia', {
  writable: true,
  value: vi.fn().mockImplementation((query: string) => ({
    matches: false,
    media: query,
    onchange: null,
    addListener: vi.fn(),
    removeListener: vi.fn(),
    addEventListener: vi.fn(),
    removeEventListener: vi.fn(),
    dispatchEvent: vi.fn(),
  })),
});
```

This ensures `matches: false` by default → `getTier()` returns `'mobile'` in tests (since neither 768px nor 1024px matchMedia will match). The deprecated `addListener`/`removeListener` stubs are included because some libraries still call them.
