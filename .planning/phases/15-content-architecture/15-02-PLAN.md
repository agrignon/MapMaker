<task type="auto">
  <name>Task 2: Fix mobile black screen — handle isMobile transition while showPreview is active</name>
  <files>src/components/Layout/SplitLayout.tsx</files>
  <action>
In SplitLayout.tsx, the useEffect at lines 147-156 handles showPreview changing while already mobile, but does NOT handle isMobile transitioning from false to true while showPreview is already true. When a user resizes from desktop to mobile width while viewing a generated model, activeTab stays at 'map', the preview pane gets display:none, and only the dark background is visible — a black screen.

The fix: Add a prevIsMobile ref and an additional condition in the existing useEffect. The updated logic:

```typescript
const prevShowPreview = useRef(showPreview);
const prevIsMobile = useRef(isMobile);
useEffect(() => {
  // Case 1: showPreview just turned on while already mobile → show preview tab
  if (showPreview && !prevShowPreview.current && isMobile) {
    setActiveTab('preview');
  }
  // Case 2: just transitioned TO mobile while showPreview is already true → show preview tab
  if (isMobile && !prevIsMobile.current && showPreview) {
    setActiveTab('preview');
  }
  // Case 3: showPreview turned off while mobile → back to map tab
  if (!showPreview && isMobile) {
    setActiveTab('map');
  }
  prevShowPreview.current = showPreview;
  prevIsMobile.current = isMobile;
}, [showPreview, isMobile]);
```

This ensures that when the viewport crosses the mobile breakpoint (768px) while a model is already generated and visible, the mobile layout immediately shows the preview tab instead of defaulting to the map tab (which hides the preview pane, producing the black screen).

Do NOT change any other part of SplitLayout.tsx. The fix is isolated to the single useEffect block.
