<task type="auto">
  <name>Task 1: Fix Overpass rate limiting — stagger road fetch after buildings</name>
  <files>src/components/Sidebar/GenerateButton.tsx</files>
  <action>
In GenerateButton.tsx handleGenerate(), lines 87-89 currently fire both fetches simultaneously:

```tsx
void fetchBuildings();
void fetchRoads();
```

Change to chain road fetch after building fetch completes (regardless of success/failure):

```tsx
void fetchBuildings().finally(() => void fetchRoads());
```

Using `.finally()` (not `.then()`) ensures roads are fetched even if buildings fail — both are optional layers, and road data should not be blocked by a building fetch error. The ~2-5 second natural delay from the building fetch completing is sufficient to avoid Overpass rate limiting.

Do NOT add a setTimeout — the sequential execution provides enough delay naturally.
Do NOT use `await` — these must remain non-blocking (terrain preview is already visible).

This fixes Root Cause 2 from 05-DIAGNOSIS.md which cascades to UAT tests 4, 5, and 6.
