    **1. Create src/lib/elevation/tiles.ts:**

    Implement these functions:

    `lonLatToTile(lon: number, lat: number, z: number): [number, number]` — Convert lon/lat to tile XYZ coordinates using the standard OSM slippy map formula:
    ```
    x = floor((lon + 180) / 360 * 2^z)
    latRad = lat * PI / 180
    y = floor((1 - ln(tan(latRad) + 1/cos(latRad)) / PI) / 2 * 2^z)
    ```

    `tileUrl(z: number, x: number, y: number, apiKey: string): string` — Return MapTiler terrain-rgb-v2 tile URL:
    `https://api.maptiler.com/tiles/terrain-rgb-v2/${z}/${x}/${y}.png?key=${apiKey}`

    `rgbToElevation(r: number, g: number, b: number): number` — Decode terrain-RGB pixel:
    `return -10000 + ((r * 256 * 256 + g * 256 + b) * 0.1)`

    `fetchTilePixels(url: string): Promise<Uint8ClampedArray>` — Fetch tile, decode with createImageBitmap + OffscreenCanvas (NOT an <img> element — avoids main thread blocking):
    ```
    fetch(url) → blob → createImageBitmap(blob) → OffscreenCanvas → ctx.drawImage → ctx.getImageData → data
    ```

    `decodeTileToElevation(pixels: Uint8ClampedArray, size: number): Float32Array` — Convert RGBA pixel array to Float32Array of elevation values. Iterate pixels in groups of 4 (RGBA), call rgbToElevation(r,g,b) for each, return Float32Array of length size*size.

    `getTileRange(sw: {lon: number, lat: number}, ne: {lon: number, lat: number}, zoom: number): { xMin: number, xMax: number, yMin: number, yMax: number, cols: number, rows: number }` — Return the range of tiles covering the bounding box at the given zoom level. Note: tile Y increases downward (north is lower Y), so yMin corresponds to ne.lat and yMax to sw.lat.

    `chooseTileZoom(sw: {lon: number, lat: number}, ne: {lon: number, lat: number}, maxTiles?: number): number` — Select optimal zoom level for the bbox. Start at zoom 12, check tile count. If tile count exceeds maxTiles (default 9 = 3x3), decrease zoom. If tile count is 1, try increasing zoom up to 14 for more detail. Return zoom in range [8, 14].

    Export all functions.

    **2. Create src/lib/elevation/stitch.ts:**

    `stitchTileElevations(tiles: { elevation: Float32Array, tileSize: number, col: number, row: number }[], cols: number, rows: number, tileSize: number): Float32Array` — Stitch multiple decoded tile elevation arrays into a single grid.

    Stitching logic:
    - Total stitched width = cols * tileSize pixels. Total height = rows * tileSize pixels.
    - For each tile, copy its elevation values into the correct position in the output array based on (col, row).
    - Adjacent tiles share a 1-pixel border with identical values. When stitching, skip the first column of each non-first horizontal tile and the first row of each non-first vertical tile to avoid duplicating border pixels. Resulting stitched size: (cols * tileSize - (cols - 1)) x (rows * tileSize - (rows - 1)).
    - Actually, per the research: take tile 0's full width (256px), each subsequent tile's columns [1..255] (skip first column). Total width = cols * 256 - (cols - 1). Same vertically.

    `resampleToMartiniGrid(elevation: Float32Array, srcWidth: number, srcHeight: number, targetSize: number): Float32Array` — Resample the stitched elevation to a (2^k+1) x (2^k+1) grid as required by martini. Use bilinear interpolation for quality. Valid target sizes: 65, 129, 257, 513. Default to 257 for single-tile, 513 for multi-tile. Return new Float32Array of targetSize * targetSize.

    `fetchElevationForBbox(sw: {lon: number, lat: number}, ne: {lon: number, lat: number}, apiKey: string): Promise<ElevationData>` — Orchestrator function:
    1. Call chooseTileZoom to pick zoom level
    2. Call getTileRange for tile grid
    3. Fetch all tiles concurrently with Promise.all (use fetchTilePixels + decodeTileToElevation from tiles.ts)
    4. Stitch with stitchTileElevations
    5. Resample to martini grid size with resampleToMartiniGrid (257 for <=1 tile, 513 for multi-tile)
    6. Compute minElevation and maxElevation from the final grid
    7. Return ElevationData { elevations, gridSize, minElevation, maxElevation }

    Import ElevationData from types/geo.ts. Import tile functions from tiles.ts.
