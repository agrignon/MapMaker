  <behavior>
  **PARSE-01 — MVT decode + property mapping:**
  - `parseOvertureTiles(tiles)` decodes each ArrayBuffer via VectorTile + Pbf
  - Overture property `height` (number) maps to `properties['height']` (string)
  - Overture property `num_floors` (number) maps to `properties['building:levels']` (string)
  - Overture property `roof_shape` (string) maps to `properties['roof:shape']` (string)
  - Overture property `roof_height` (number) maps to `properties['roof:height']` (string)
  - All output features have `properties['building'] = 'yes'`
  - Features with no height data still produce valid BuildingFeature (building=yes triggers fallback)
  - Single tile with 3 features returns 3 BuildingFeature entries
  - Multiple tiles accumulate features into a single flat array
  - Tiles with no "building" layer are silently skipped (return [])
  - Empty Map input returns []

  **PARSE-02 — MultiPolygon flattening:**
  - Feature with geometry.type === 'MultiPolygon' and 2 sub-polygons produces 2 separate BuildingFeature entries
  - Each sub-polygon gets the same mapped properties
  - Mixed tile with Polygon + MultiPolygon features produces correct total count

  **PARSE-03 — Winding normalization:**
  - CW outer ring (negative computeSignedArea) is reversed to CCW in output BuildingFeature.outerRing
  - Already-CCW outer ring (positive computeSignedArea) is left unchanged (defensive)
  - Hole rings from toGeoJSON are passed through unchanged (already CCW)
  - After normalization, computeSignedArea(output.outerRing) > 0 for all features

  **PARSE-04 — Area filter:**
  - Feature with footprint < 15 m2 is excluded from output
  - Feature with footprint >= 15 m2 is included in output
  - Feature with footprint exactly 15 m2 is included (threshold is >=, not >)
  - Degenerate rings (< 3 unique vertices after closing vertex removal) are excluded
  </behavior>
