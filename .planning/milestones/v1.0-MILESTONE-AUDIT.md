## Cross-Phase Integration

### Integration Findings

**1. locationName not wired from geocoding (Phase 1 → Phase 2)**
- `setLocationName` exists in `mapStore.ts` and is consumed in `ExportPanel.tsx` for filename generation
- `SearchOverlay.tsx` never calls `setLocationName` — filenames always fall back to coordinates
- Impact: Cosmetic — feature works but users get `terrain-46.85--121.73.stl` instead of `mount-rainier-terrain.stl`
- Affected: EXPT-04 (degraded, not broken)

**2. stl.ts is dead code (Phase 1)**
- `metersToMillimeters` and `bboxToMM` are tested (4 tests) but never imported in production code
- Phase 2 computes mm scaling via `horizontalScale = widthMM / geographicWidthM` directly
- Impact: None — dead code, tests pass but module is unused

**3. Production build fails (cross-phase)**
- `npm run build` exits with 4 TypeScript errors:
  - `PreviewControls.tsx:2` — unused `THREE` import
  - `walls.test.ts:131` — unused variable
  - `terrain.ts:7` — missing `@mapbox/martini` type declaration
  - `terrain.ts:69` — unused `geographicDepthM`
- `npx tsc --noEmit` passes (vitest path uses different tsconfig)
- Impact: Cannot produce production build artifact
