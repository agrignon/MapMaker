<objective>
Create the Overture Maps PMTiles fetch module that retrieves raw building tile data for any
bounding box, with 5-second timeout, AbortSignal cancellation, and silent fallback to empty results
on any error.

Purpose: Provides the data access layer for Overture building gap-fill. Phase 11 will decode the
raw ArrayBuffer tiles into BuildingFeature format; this phase only fetches and returns raw bytes.

Output: `src/lib/overture/` module with public `fetchOvertureTiles()` API, constants with documented
URL rotation, and comprehensive unit tests.
</objective>

  <behavior>
    TDD behavior specifications (write tests FIRST, then implement):

    **Tile enumeration (bboxToTileKeys):**
    - bboxToTileKeys({ sw: {lon: -74.01, lat: 40.70}, ne: {lon: -73.97, lat: 40.75} }) returns an array of [x, y, z] tuples at zoom 14
    - All returned tuples have z === 14
    - A tiny bbox (single point) returns at least 1 tile
    - A bbox spanning multiple tiles returns multiple tiles

    **Fetch with timeout (fetchOvertureTiles):**
    - On success: returns { tiles: Map with at least 1 entry, available: true }
    - On network error: returns { tiles: empty Map, available: false } — does NOT throw
    - On AbortError (timeout): returns { tiles: empty Map, available: false } — does NOT throw
    - On caller abort signal: returns { tiles: empty Map, available: false } — does NOT throw
    - On empty tiles (all getZxy return undefined): returns { tiles: empty Map, available: true } — empty is valid data, NOT a failure
    - console.warn is called on any error path with descriptive message

    **Constants:**
    - OVERTURE_BUILDINGS_PMTILES_URL contains 'tiles.overturemaps.org' and '2026-02-18.0'
    - OVERTURE_BUILDING_LAYER === 'building' (singular, NOT 'buildings')
    - OVERTURE_FETCH_ZOOM === 14
    - OVERTURE_FETCH_TIMEOUT_MS === 5000
    - OVERTURE_STAC_CATALOG_URL contains 'stac.overturemaps.org'
  </behavior>

    **Step 3: Create src/lib/overture/tiles.ts**
    Two internal functions:

    `bboxToTileKeys(bbox: BoundingBox): [number, number, number][]`
    - Convert BoundingBox to GeoJSON Polygon (ring: [w,s],[e,s],[e,n],[w,n],[w,s])
    - Call `cover.tiles(polygon, { min_zoom: 14, max_zoom: 14 })`
    - Return the result as-is (array of [x, y, z] tuples)

    `fetchTilesFromArchive(bbox: BoundingBox, signal: AbortSignal): Promise<Map<string, ArrayBuffer>>`
    - Create `new PMTiles(OVERTURE_BUILDINGS_PMTILES_URL)`
    - Get tile list via `bboxToTileKeys(bbox)`
    - `Promise.all()` over tiles: for each `[x, y, z]`, call `archive.getZxy(z, x, y, signal)` — NOTE z first in getZxy
    - Collect non-undefined responses into `Map<string, ArrayBuffer>` keyed by `"z/x/y"`
    - Let errors propagate (caller handles)

    **Step 4: Create src/lib/overture/index.ts**
    Public API:

    ```typescript
    export interface OvertureResult {
      tiles: Map<string, ArrayBuffer>;
      available: boolean;
    }

    export async function fetchOvertureTiles(
      bbox: BoundingBox,
      callerSignal?: AbortSignal
    ): Promise<OvertureResult>
    ```

    - Create AbortController
    - Set 5-second timeout via setTimeout → controller.abort()
    - If callerSignal provided, propagate abort to controller
    - try: call fetchTilesFromArchive(bbox, controller.signal) → return { tiles, available: true }
    - catch: console.warn('[Overture] fetch failed, falling back to OSM-only:', err) → return { tiles: new Map(), available: false }
    - finally: clearTimeout(timeoutId)
    - Handle edge case: if result map is empty but no error was thrown, still return available: true (empty valid data)

    Mock `@mapbox/tile-cover` to return deterministic tile lists.
    Use `vi.spyOn(console, 'warn')` to verify warning logs on error paths.
    Test AbortController by making the mock `getZxy` reject with AbortError.
