  <action>
    Create `src/lib/mesh/terrain.ts` implementing:

    **`elevationToColor(t: number): [number, number, number]`** — Map normalized elevation (0-1) to RGB hypsometric tint stops:
    - 0.0-0.3: dark green (#2d6a2d) to yellow-green (#6b8c42) — lowlands/valleys
    - 0.3-0.65: yellow-green (#6b8c42) to brown (#8b6a3e) — mid elevations
    - 0.65-1.0: brown (#8b6a3e) to white (#ffffff) — peaks/snow
    Use linear interpolation within each range. Return [r, g, b] each in 0-1 range (for Three.js Color).

    **`buildTerrainGeometry(elevationData: ElevationData, params: TerrainMeshParams): THREE.BufferGeometry`**

    TerrainMeshParams interface:
    ```typescript
    interface TerrainMeshParams {
      widthMM: number;      // target physical width
      depthMM: number;      // target physical depth
      exaggeration: number; // slider value (default 1.5)
      minHeightMM: number;  // TERR-03 minimum (default 5)
      maxError: number;     // RTIN error threshold (default 5, lower = more triangles)
    }
    ```

    Implementation steps:
    1. Create Martini instance: `new Martini(elevationData.gridSize)`. **CRITICAL:** gridSize must be 2^k+1 (e.g., 257 or 513). Martini constructor takes the grid size, NOT tile pixel size.
    2. Create tile: `martini.createTile(elevationData.elevations)`
    3. Get mesh: `tile.getMesh(maxError)` returns `{ vertices, triangles }` where vertices is [x0,y0, x1,y1, ...] and triangles is [i0,i1,i2, ...] indices.
    4. Compute elevation range: `maxElev - minElev` from elevationData.
    5. Compute Z scaling: If elevation range > 0, zScale = max(exaggeration, minHeightMM / elevRange_MM). If range is 0 (perfectly flat), set all Z to minHeightMM.
    6. Build positions Float32Array (vertices.length/2 * 3):
       - X = (vx / (gridSize-1)) * widthMM — centered at widthMM/2
       - Y = (vy / (gridSize-1)) * depthMM — centered at depthMM/2 (note: Y is depth, Z is up)
       - Z = (elevation[vy * gridSize + vx] - minElev) * zScale
       Center the mesh: subtract widthMM/2 from X, depthMM/2 from Y. This makes the mesh origin at the center base, which is natural for orbiting.
    7. Build vertex colors Float32Array (same vertex count * 3):
       - For each vertex, compute `t = (elevation - minElev) / elevRange` (0-1)
       - Call elevationToColor(t) to get [r,g,b]
    8. Create THREE.BufferGeometry:
       - `geometry.setAttribute('position', new THREE.BufferAttribute(positions, 3))`
       - `geometry.setAttribute('color', new THREE.BufferAttribute(colors, 3))`
       - `geometry.setIndex(new THREE.BufferAttribute(new Uint32Array(triangles), 1))` — use Uint32Array for the index (martini returns Uint32Array)
       - `geometry.computeVertexNormals()`
    9. Return geometry.

    **`updateTerrainElevation(geometry: THREE.BufferGeometry, elevationData: ElevationData, params: TerrainMeshParams): void`** — Update Z positions and colors in-place when exaggeration changes (avoids full remesh).
    1. Get position attribute: `geometry.getAttribute('position') as THREE.BufferAttribute`
    2. Recompute Z for each vertex using new exaggeration
    3. Set `positionAttribute.needsUpdate = true`
    4. Recompute normals: `geometry.computeVertexNormals()`
    5. Colors stay the same (exaggeration doesn't change elevation mapping).

    Import `THREE` from 'three', `Martini` from '@mapbox/martini', `ElevationData` from types/geo.ts.

    **Anti-patterns to avoid:**
    - Do NOT re-create the Martini instance on each exaggeration change. Only the Z positions change.
    - Do NOT forget `computeVertexNormals()` after any position update — the mesh will shade incorrectly.
    - Do NOT use `new Martini(256)` — always use gridSize (257 or 513), which is 2^k+1.
  </action>
