<objective>
Fix the tile boundary seam bug in stitchTileElevations. The function falsely assumes adjacent MapTiler terrain-rgb-v2 tiles share a 1-pixel border overlap and deduplicates it. Standard XYZ raster tiles are exactly 256x256 with NO overlap. This causes wrong grid dimensions, dropped elevation data at tile boundaries, and zero-elevation strips at east/south edges — producing visible cliff-like seams in the terrain mesh.

Purpose: Close both remaining UAT gaps (tile boundary seams + geographic orientation distortion). Both share this single root cause in stitch.ts.
Output: Corrected stitch.ts with simple concatenation stitching, plus regression tests proving no boundary discontinuities.
</objective>
