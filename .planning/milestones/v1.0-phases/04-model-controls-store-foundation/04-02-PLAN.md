**CollapsibleSection.tsx — Reusable collapsible section component:**

Create a component with props:
- `label: string` — section title (e.g., "Terrain", "Buildings")
- `summary?: string` — shown when collapsed (e.g., "1.5x exag, 3mm base")
- `toggle?: { checked: boolean; onChange: (v: boolean) => void; disabled?: boolean }` — optional layer toggle pill in the header row
- `defaultOpen?: boolean` — initial expand state (default true)
- `children: React.ReactNode` — section body (hidden when collapsed)

Layout:
- Header row: clickable to expand/collapse. Left side: chevron (▸/▾) + label. Right side: optional toggle switch pill. When collapsed, show `summary` text below the label in smaller muted text.
- The toggle pill and expand/collapse are SEPARATE click targets. Clicking the label/chevron toggles expand. Clicking the toggle pill changes layer visibility via onChange. Per Research Open Question #3 recommendation.
- Toggle switch visual: small pill with `role="switch"` and `aria-checked`. Active = blue (#2563eb) background, inactive = gray (#374151). Disabled = 50% opacity + cursor: not-allowed + no onClick.
- Collapse state: local `useState(defaultOpen ?? true)` — ephemeral UI state, NOT in Zustand. Per Research anti-pattern guidance.
- Style: match existing dark glass aesthetic (rgba backgrounds, #e5e7eb text, #6b7280 muted, border-bottom: 1px solid rgba(255,255,255,0.08) between sections).

**ModelSizeSection.tsx — Model dimensions and unit toggle:**

This component replaces the "Model Dimensions" section currently in ExportPanel.tsx. Per user decision: "A top-level 'Model Size' section above all layer sections for dimensions and unit toggle."

Read from store: `targetWidthMM`, `targetDepthMM`, `targetHeightMM`, `units`, `dimensions`, `elevationData`, `exaggeration`.
Actions: `setTargetWidth`, `setTargetHeightMM`, `setUnits`.

Layout:
1. **Unit toggle** — Segmented control (mm | in) at the top. Two buttons in a pill. Active segment = blue (#2563eb) bg. Per user decision: "Segmented control (mm | in) for unit toggle, inline with dimension inputs." Uses `setUnits` action.

2. **Width (X) input** — Free-form number input. Label: "Width (X)". Displays value in current unit: `units === 'in' ? targetWidthMM / 25.4 : targetWidthMM`. On blur, converts back to mm: `units === 'in' ? parseFloat(value) * 25.4 : parseFloat(value)`. Calls `setTargetWidth(mmValue)` which auto-calculates depth. Use local string state for keystroke buffering (like existing ExportPanel pattern), commit to store on blur.

3. **Depth (Y) display** — READ-ONLY computed value. Shows `targetDepthMM` converted to display unit. Label: "Depth (Y)". Per user decision: "Width (X) input with auto-calculated depth (Y) that maintains bbox aspect ratio." Display as a styled span or disabled input, not an editable field. Per Research Pitfall 2: depth is auto-calculated, must not cache a stale local state.

4. **Height (Z) input** — Free-form number input. Label: "Height (Z)". When `targetHeightMM === 0`: show auto-calculated value as placeholder text. Auto value = `(elevRange * exaggeration * zScale) + basePlateThicknessMM` where `zScale = targetWidthMM / (dimensions?.widthM ?? 1) / 1000` (matching terrain.ts pattern). When user types a value, store it via `setTargetHeightMM(mmValue)`. When user clears to empty, revert to 0 (auto). Per Research Open Question #1: show auto as default, allow override, clear reverts to auto.

5. **Summary line** — Below inputs, show computed physical size: "150 × 112 × 23.4 mm" or "5.91 × 4.41 × 0.92 in" depending on unit. Use muted text (#6b7280, 11px).

Conversion constants: `const MM_PER_INCH = 25.4;`. Helper functions:
- `toDisplayUnit(valueMM: number, units: 'mm' | 'in'): number` — returns `units === 'in' ? valueMM / MM_PER_INCH : valueMM`
- `toStoreMM(displayValue: number, units: 'mm' | 'in'): number` — returns `units === 'in' ? displayValue * MM_PER_INCH : displayValue`

Per user decision: "Switching units converts existing values (150mm → 5.91in), not reset." Since we store in mm and convert on display, this happens automatically — no stored value mutation needed.

Input validation: Width min=10mm/0.4in, max=500mm/19.7in. Height min=0 (auto), max=200mm/7.9in. Step: 1mm or 0.1in.

Style: Match existing dark glass aesthetic. Input style matches ExportPanel's inputStyle pattern: `backgroundColor: '#1f2937', border: '1px solid #374151', borderRadius: '6px', color: '#e5e7eb', padding: '6px 10px', fontSize: '13px'`.
