1. Create `src/lib/mesh/buildingSolid.ts`:
   - `unionBuildingsWithTerrain(terrainGeometry: THREE.BufferGeometry, buildingsGeometry: THREE.BufferGeometry): THREE.BufferGeometry`
   - Import `{ ADDITION, Brush, Evaluator }` from `'three-bvh-csg'`
   - Create Brush from terrain geometry (must be non-indexed for CSG — call `toNonIndexed()` if indexed)
   - Create Brush from buildings geometry (already non-indexed from mergeGeometries)
   - Call `brush.updateMatrixWorld()` on both
   - Create Evaluator and call `evaluator.evaluate(terrainBrush, buildingsBrush, ADDITION)`
   - Return `result.geometry`
   - PERFORMANCE NOTE from research: three-bvh-csg on terrain-scale meshes is unvalidated. Wrap in try/catch. If CSG fails or takes > 10 seconds (use a timeout check), fall back to simple `mergeGeometries([terrainSolid, buildingsGeometry])` with a console.warn. The fallback may produce non-manifold edges at terrain-building seams but is better than a broken export.
   - Export a `mergeTerrainAndBuildings(terrainSolid, buildingsGeo): THREE.BufferGeometry` convenience function that attempts CSG union first, falls back to merge.

2. Update `src/components/Preview/ExportPanel.tsx`:
   - In the `handleExport()` function, after building the terrain solid mesh (`buildSolidMesh`), check if building features exist in store
   - If `buildingFeatures` and `buildingFeatures.length > 0`:
     a. Add a new export step: `setExportStatus('building', 'Building buildings mesh...')`
     b. Call `buildAllBuildings(buildingFeatures, bbox, elevationData, params)` to get merged building geometry
     c. Convert building geometry base elevation to use the same zScale as the terrain (the same scale factor used in terrain.ts)
     d. `setExportStatus('building', 'Merging terrain and buildings...')`
     e. Call `mergeTerrainAndBuildings(terrainSolid, buildingsGeometry)` to get the final combined solid
     f. Use the combined solid for validation and STL export (replacing the terrain-only solid)
   - If no building features: export terrain-only as before (backward compatible)
   - Update progress bar step text to reflect building steps
   - Read `buildingFeatures` from store via `useMapStore`

3. Update `src/lib/export/stlExport.ts`:
   - Update `generateFilename()`: if buildings are present, change suffix from `-terrain.stl` to `-terrain-buildings.stl`
   - Add optional `hasBuildings: boolean` parameter to generateFilename
   - Coordinate-based fallback also gets updated suffix

4. Create `src/lib/mesh/__tests__/buildingSolid.test.ts`:
   - Test that `unionBuildingsWithTerrain()` produces a manifold mesh for a simple scenario:
     a. Construct a minimal terrain solid: a flat 10x10x2 box using `THREE.BoxGeometry(10, 10, 2)`
     b. Construct a minimal building: a 2x2x5 box using `THREE.BoxGeometry(2, 2, 5)` positioned at Y=1 (sitting on top of terrain)
     c. Call `unionBuildingsWithTerrain(terrainGeo, buildingGeo)`
     d. Validate the result with `validateMesh()` from `src/lib/export/validate.ts` — assert it returns `isManifold: true`
   - Test the merge fallback path: verify `mergeTerrainAndBuildings()` returns geometry (even if non-manifold) when CSG fails
   - This test validates the CSG pipeline before the human gate, catching manifold issues automatically

5. COORDINATE CONSISTENCY (already handled):
   - The merge.ts `buildAllBuildings()` function (from Plan 03-01) already accepts `BuildingGeometryParams` which includes `exaggeration` and `minElevationM`, and computes zScale identically to terrain.ts: `horizontalScale = widthMM / geographicWidthM`, `zScale = horizontalScale * exaggeration`, `baseZmm = (elevationM - minElevationM) * zScale`.
   - The ExportPanel must pass these params correctly when calling `buildAllBuildings()` — read `exaggeration` from store and `minElevation` from `elevationData.minElevation`.
   - No modifications to merge.ts are needed here — the zScale consistency was established in Plan 03-01.
