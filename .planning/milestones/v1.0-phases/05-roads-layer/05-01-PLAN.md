**Step 1: Create `src/lib/roads/roadMesh.ts`**

Export `buildRoadGeometry(features: RoadFeature[], bbox: BoundingBox, elevData: ElevationData, params: RoadGeometryParams): THREE.BufferGeometry | null`

**Constants (Claude's discretion per CONTEXT.md):**
```typescript
// Road widths in mm (model space at 150mm model width)
const ROAD_WIDTH_MM: Record<RoadTier, number> = {
  highway: 1.8,
  main: 1.2,
  residential: 0.7,
};

// Road depth/height offsets in mm ظ¤ within the 0.5-1.0mm range for deepest
const ROAD_DEPTH_MM: Record<RoadTier, number> = {
  highway: 1.0,     // full depth (locked decision)
  main: 0.6,        // ~60% (locked decision)
  residential: 0.3, // ~30% (locked decision)
};

const ROAD_COLOR = '#555555'; // dark gray (locked decision: #555 range)
```

Export these constants so RoadMesh.tsx can access ROAD_COLOR.

**Algorithm for each feature:**

1. **Compute zScale** ظ¤ MUST match terrain.ts and buildings/merge.ts exactly:
   ```typescript
   const horizontalScale = widthMM / geographicWidthM;
   const elevRange = elevData.maxElevation - elevData.minElevation;
   let zScale: number;
   if (elevRange === 0) {
     zScale = horizontalScale * exaggeration;
   } else if (targetReliefMM && targetReliefMM > 0) {
     zScale = (targetReliefMM / elevRange) * exaggeration;
   } else {
     const MIN_HEIGHT_MM = 5;
     const naturalHeightMM = elevRange * horizontalScale * exaggeration;
     zScale = naturalHeightMM < MIN_HEIGHT_MM ? MIN_HEIGHT_MM / elevRange : horizontalScale * exaggeration;
   }
   ```
   Compute zScale ONCE before the loop, not per feature.

2. **Project coordinates to local mm space** ظ¤ same pipeline as buildings/merge.ts `projectRingToMM`:
   ```typescript
   function projectCenterlineToMM(coords: [number, number][], centerUTM, horizontalScale): [number, number][] {
     return coords.map(([lon, lat]) => {
       const utm = wgs84ToUTM(lon, lat);
       return [(utm.x - centerUTM.x) * horizontalScale, (utm.y - centerUTM.y) * horizontalScale];
     });
   }
   ```

3. **Compute per-vertex terrain Z** ظ¤ for each original [lon, lat] coordinate, call `sampleElevationAtLonLat(lon, lat, bbox, elevData)` and compute `(elevM - minElevationM) * zScale`.

4. **Compute style offset per tier:**
   - `recessed`: offset = `-ROAD_DEPTH_MM[tier]`
   - `raised`: offset = `+ROAD_DEPTH_MM[tier]`
   - `flat`: offset = `0`

5. **Handle bridges:** For `isBridge === true`, linearly interpolate Z between the first and last terrain Z values (no style offset for bridges; they span above terrain). Add a bridge lift of `+ROAD_DEPTH_MM[tier] * 2` (double the road depth) so bridges visually float above terrain.

6. **Call extrudePolyline** from geometry-extrude:
   ```typescript
   import { extrudePolyline } from 'geometry-extrude';

   const ribbonDepth = ROAD_DEPTH_MM[feature.tier]; // thin ribbon
   const { position, indices, normal } = extrudePolyline(
     [projected2D],  // MUST wrap in array ظ¤ MultiLineString format (Pitfall 2)
     { lineWidth: ROAD_WIDTH_MM[feature.tier], depth: ribbonDepth, miterLimit: 2 }
   );
   ```

7. **Assign per-vertex terrain Z** ظ¤ Critical step (Pitfall 1). extrudePolyline produces flat geometry at Z=0..ribbonDepth. Post-process the `position` Float32Array:
   - For each vertex `(vx, vy, vz)` in the position array, find the nearest centerline segment and compute the interpolated terrain Z at that XY location
   - Use a simple approach: for each vertex, find the closest projected 2D centerline point by iterating segments and projecting the vertex onto each segment. Use the interpolated terrain Z + style offset at that location
   - The vertex's local z (0 to ribbonDepth) represents the ribbon thickness ظ¤ add it to the terrain Z + style offset
   - Implementation: iterate centerline segments, project vertex XY onto each segment, find nearest, interpolate terrain Z between segment endpoints proportionally

   Helper function:
   ```typescript
   function assignTerrainZ(
     position: Float32Array,
     centerline2D: [number, number][],
     terrainZs: number[],
     styleOffset: number,
     ribbonDepth: number
   ): void
   ```

8. **Build Three.js BufferGeometry** from the post-processed arrays:
   ```typescript
   const geo = new THREE.BufferGeometry();
   geo.setAttribute('position', new THREE.BufferAttribute(new Float32Array(position), 3));
   geo.setAttribute('normal', new THREE.BufferAttribute(new Float32Array(normal), 3));
   geo.setIndex(new THREE.BufferAttribute(new Uint32Array(indices), 1));
   ```
   Note: Use `new Float32Array(position)` to copy the TypedArray ظ¤ geometry-extrude may reuse internal buffers. Use Uint32Array for indices (road meshes can exceed 65535 vertices).

9. **Skip roads with < 2 points** after projection.

10. **Merge all road geometries** using `mergeGeometries` from `three/addons/utils/BufferGeometryUtils.js`. Strip `uv` attribute before merge (Pitfall 7 ظ¤ geometry-extrude returns UV but terrain/buildings lack it). Call `merged.computeVertexNormals()` after merge. Dispose individual geometries.

11. Return `null` if no geometries produced (zero roads in area).
